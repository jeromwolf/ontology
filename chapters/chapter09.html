9.1 Protégé란 무엇인가?

Protégé는 스탠포드 대학에서 개발한 오픈소스 온톨로지 에디터로, 전 세계적으로 가장 널리 사용되는 온톨로지 개발 도구입니다.

핵심 개념

Protégé: OWL 온톨로지를 시각적으로 생성, 편집, 관리할 수 있는 통합 개발 환경(IDE)

Protégé의 특징

직관적인 UI

그래픽 인터페이스로 복잡한 온톨로지도 쉽게 편집

  • 드래그 앤 드롭 지원

  • 표준 준수 검사

9.3 클래스 작업하기

클래스 생성과 편집

새 클래스 생성

  1. Classes 탭 선택
  2. 상위 클래스 선택 (기본: owl:Thing)
  3. Add subclass 버튼 클릭 또는 Ctrl+Alt+C
  4. 클래스 이름 입력
예시: 
Thing
└── Vehicle
    ├── Car
    ├── Bicycle
    └── Airplane

클래스 설명 추가

SubClass Of

필요조건 정의

Car SubClassOf hasWheels exactly 4
Equivalent To

필요충분조건 정의

Parent EquivalentTo Person and hasChild some Person
Disjoint With

상호배타적 클래스

Male DisjointWith Female

클래스 표현식 만들기

교집합 (Intersection)

Manchester 구문: Person and hasAge some int[>=18]

의미: 18세 이상인 사람

합집합 (Union)

Manchester 구문: Cat or Dog

의미: 고양이 또는 개

여집합 (Complement)

Manchester 구문: not Vegetarian

의미: 채식주의자가 아닌 것

속성 제한

Manchester 구문: hasChild only Female

의미: 딸만 있는 경우

9.4 속성 작업하기

Object Properties

속성 생성

  1. Object properties 탭 선택
  2. Add sub property 클릭
  3. 속성 이름 입력 (예: hasParent)
  4. 도메인과 레인지 설정

속성 특성 설정

Functional

하나의 값만 가능

hasBirthDate
Inverse Functional

값이 유일함

hasSSN
Transitive

이행적 관계

hasAncestor
Symmetric

대칭적 관계

marriedTo
Asymmetric

비대칭적 관계

parentOf
Reflexive

반사적 관계

knows

Data Properties

데이터 속성 예시

  • hasName: xsd:string
  • hasAge: xsd:integer
  • hasBirthDate: xsd:dateTime
  • hasEmail: xsd:string[pattern "^.+@.+$"]

데이터타입 제약

hasAge some xsd:integer[>= 0, <= 150]
hasGPA some xsd:float[>= 0.0, <= 4.0]
hasStatus some {"active", "inactive", "pending"}

9.5 개체(Individuals) 작업

개체 생성과 관리

개체 생성 단계

  1. Individuals by class 뷰 선택
  2. 해당 클래스 선택
  3. Add individual 클릭
  4. 개체 이름 입력
  5. 속성 값 추가

예시: 사람 개체

Individual: john_doe
    Types: Person
    Facts:
        hasName "John Doe"
        hasAge 35
        hasEmail "john@example.com"
        worksFor acme_corp
        knows jane_doe

단언(Assertion) 유형

Object Property Assertions

개체 간의 관계

john_doe knows jane_doe
Data Property Assertions

개체의 데이터 값

john_doe hasAge 35
Same/Different Individuals

개체 동일성/상이성

john SameAs john_doe

9.6 추론기 활용하기

추론기 설정과 실행

추론기 시작하기

  1. Reasoner 메뉴 열기
  2. 추론기 선택 (예: HermiT)
  3. Start reasoner 클릭 또는 Ctrl+R
  4. 추론 결과 확인

추론 유형

클래스 계층 추론

암묵적 상하위 관계 발견

정의: Mother ≡ Female ⊓ Parent

추론: Mother ⊑ Person

개체 분류

개체의 추가 타입 추론

사실: mary hasChild tom

추론: mary is Parent

일관성 검사

논리적 모순 검출

🔴 빨간색 클래스: 불일치

🟡 노란색 표시: 추론된 내용

추론기별 특징

추론기 장점 단점 사용 시기
HermiT 정확성, 디버깅 속도 개발/디버깅
Pellet 균형잡힌 성능 업데이트 중단 일반 용도
ELK 매우 빠름 EL만 지원 대규모 분류
FaCT++ 빠른 속도 제한된 기능 성능 중요

9.7 시각화와 플러그인

OntoGraf - 온톨로지 시각화

OntoGraf 사용법

  1. Window → Views → Ontology views → OntoGraf
  2. 시각화할 클래스 선택
  3. Show 버튼 클릭
컨트롤
  • 드래그: 노드 이동
  • 휠: 확대/축소
  • 더블클릭: 노드 확장/축소
  • 우클릭: 컨텍스트 메뉴

유용한 플러그인

Ontology Metrics

온톨로지 통계 분석

  • 클래스/속성/개체 수
  • DL 표현력
  • 복잡도 메트릭
OWL Doc

온톨로지 문서 자동 생성

  • HTML 문서화
  • 클래스 다이어그램
  • API 문서
Cellfie

스프레드시트에서 온톨로지 생성

  • Excel 임포트
  • 매핑 규칙
  • 일괄 변환

9.8 실전 프로젝트: 도서관 온톨로지

단계별 구축 가이드

1단계: 프로젝트 설정

  1. File → New → 온톨로지 IRI 설정
  2. Prefix 추가: lib → http://example.org/library#
  3. 메타데이터 추가 (제목, 설명, 버전)

2단계: 클래스 계층구조

Thing
├── Resource
│   ├── PhysicalResource
│   │   ├── Book
│   │   ├── Magazine
│   │   └── DVD
│   └── DigitalResource
│       ├── EBook
│       └── AudioBook
├── Person
│   ├── Member
│   │   ├── Student
│   │   └── Faculty
│   ├── Author
│   └── Librarian
└── Subject
    ├── Fiction
    └── NonFiction

3단계: 속성 정의

Object Properties
  • borrowedBy (도메인: Resource, 레인지: Member)
  • writtenBy (도메인: Book, 레인지: Author)
  • hasSubject (도메인: Resource, 레인지: Subject)
Data Properties
  • hasISBN (도메인: Book, 레인지: string)
  • hasTitle (도메인: Resource, 레인지: string)
  • publishedYear (도메인: Resource, 레인지: integer)

4단계: 제약사항 추가

  • Book: hasISBN exactly 1
  • Member: borrowedBy max 5 Resource
  • hasISBN: Functional
  • Student DisjointWith Librarian

5단계: 개체 생성

Individual: book_001
    Types: Book
    Facts:
        hasTitle "온톨로지 입문"
        hasISBN "978-89-123-4567-8"
        writtenBy author_kim
        hasSubject ontology_subject
        publishedYear 2024

6단계: 추론과 검증

  1. HermiT 추론기 시작
  2. 일관성 검사
  3. 추론된 계층구조 확인
  4. DL Query로 테스트

🎯 실습 과제

과제 1: 학교 온톨로지 구축

Protégé를 사용하여 다음 요구사항을 만족하는 학교 온톨로지를 구축하세요:

  1. 학생, 교사, 과목, 교실 클래스 정의
  2. 수강(enrolledIn), 가르침(teaches) 관계 정의
  3. 학생은 최대 7과목 수강 가능 제약
  4. 각 과목은 반드시 1명의 교사가 담당
  5. 최소 10개의 개체 생성
  6. 추론기로 일관성 검증

다음 플러그인을 설치하고 활용해보세요:

  1. OntoGraf로 온톨로지 시각화
  2. Ontology Metrics로 복잡도 분석
  3. DL Query로 복잡한 질의 실행
