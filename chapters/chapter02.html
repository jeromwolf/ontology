2.1 클래스(Class): 개념의 범주화

클래스는 온톨로지의 가장 기본적인 구성 요소로, 유사한 특징을 가진 개체들의 집합을 나타냅니다.

핵심 개념

클래스(Class): 공통된 속성을 가진 개체들의 범주 또는 타입

클래스의 특징

1. 추상화

개별 개체들의 공통 특성을 추출하여 일반화

예시:
• 사람 클래스: 홍길동, 김철수, 이영희의 공통 특성
• 자동차 클래스: 소나타, K5, 그랜저의 공통 특성

2. 계층구조

상위 클래스와 하위 클래스의 관계로 조직화

2.2 인스턴스(Instance): 구체적인 개체

인스턴스는 클래스의 구체적인 예시로, 실제 세계에 존재하는 개별 개체를 나타냅니다.

인스턴스와 클래스의 관계

클래스: 책

속성: 제목, 저자, ISBN, 출판사

↓ is-a 관계

ISBN: "978-89-374-8322-4"
출판사: "민음사"

인스턴스 생성 규칙

1. 유일성

각 인스턴스는 고유한 식별자를 가져야 함

책_사피엔스_001 rdf:type 책 .
책_사피엔스_002 rdf:type 책 . # 다른 판본

2. 타입 일관성

인스턴스는 해당 클래스의 모든 속성을 만족해야 함

✅ 올바른 예: 모든 필수 속성 포함
❌ 잘못된 예: ISBN 없는 책 인스턴스

3. 명명 규칙

의미 있고 일관된 이름 사용

• 좋은 예: Book_Sapiens_2015_KR
• 나쁜 예: b1, temp_book, 책123

2.3 속성(Property): 특성과 관계

속성은 클래스나 인스턴스의 특징을 설명하고, 개체 간의 관계를 정의합니다.

속성의 종류

1. 객체 속성 (Object Property)

개체와 개체 사이의 관계

저자관계
책 —[저자]→ 사람
소속관계
직원 —[근무처]→ 회사
위치관계
건물 —[위치]→ 도시

2. 데이터 속성 (Data Property)

개체의 값이나 리터럴 데이터

텍스트 속성
  • 이름: "홍길동"
  • 제목: "온톨로지 입문"
  • 설명: "이 책은..."
숫자 속성
  • 나이: 25
  • 가격: 15000
  • 페이지수: 350
날짜 속성
  • 생년월일: 1990-01-01
  • 출판일: 2024-03-15
  • 등록일: 2024-04-01

속성의 특성

기능적 속성 (Functional)

하나의 주체는 하나의 값만 가질 수 있음

• 생년월일 (사람당 하나)
• ISBN (책당 하나)
• 주민등록번호 (사람당 하나)

역기능적 속성 (Inverse Functional)

하나의 값은 하나의 주체만 가질 수 있음

• 주민등록번호 (고유값)
• 이메일 주소 (중복 불가)
• 사원번호 (회사 내 고유)

대칭적 속성 (Symmetric)

A→B이면 B→A도 성립

• 친구 관계
• 형제 관계
• 동료 관계

이행적 속성 (Transitive)

A→B이고 B→C이면 A→C도 성립

• 조상 관계
• 상위 개념 관계
• 위치 포함 관계

2.4 제약사항(Constraint): 규칙과 제한

제약사항은 온톨로지의 일관성과 정확성을 보장하기 위한 규칙입니다.

주요 제약사항 유형

1. 카디널리티 제약

관계의 개수 제한

제약 유형 설명 예시
정확히 1 반드시 하나 사람은 정확히 1개의 생년월일을 가짐
최소 1 하나 이상 책은 최소 1명의 저자를 가짐
최대 2 둘 이하 사람은 최대 2명의 생물학적 부모를 가짐
0..n 제한 없음 사람은 0개 이상의 자녀를 가질 수 있음

2. 값 제약

속성 값의 범위나 형식 제한

범위 제약 나이: 0 ≤ 값 ≤ 150
평점: 0.0 ≤ 값 ≤ 5.0
할인율: 0% ≤ 값 ≤ 100%
형식 제약 이메일: *@*.*
전화번호: 010-****-****
우편번호: #####
열거형 제약 성별: {남, 여, 기타}
혈액형: {A, B, O, AB}
요일: {월, 화, 수, 목, 금, 토, 일}

3. 논리적 제약

클래스 간의 관계 규칙

분리 제약 (Disjoint)

두 클래스가 공통 인스턴스를 가질 수 없음

남성 ⊥ 여성
동치 제약 (Equivalent)

두 클래스가 동일한 인스턴스 집합을 가짐

부모 ≡ (아버지 ∪ 어머니)
포함 제약 (Subclass)

한 클래스가 다른 클래스에 포함됨

학생 ⊆ 사람

2.5 추론(Reasoning): 지식의 확장

추론은 명시적으로 저장되지 않은 지식을 논리적으로 도출하는 과정입니다.

추론의 유형

1. 분류 추론

개체가 어떤 클래스에 속하는지 결정

전제:
• 포유류는 젖을 먹여 새끼를 키운다
• 고래는 젖을 먹여 새끼를 키운다
결론:
→ 고래는 포유류이다

2. 속성 추론

속성 값을 논리적으로 도출

전제:
• 철수의 아버지는 영수이다
• 영수의 아버지는 민수이다
• '할아버지' = '아버지의 아버지'
결론:
→ 철수의 할아버지는 민수이다

3. 일관성 검증

모순되는 정보 검출

모순 발견:
• 사람 A는 미혼이다
• 사람 A는 배우자가 있다
오류:
→ 논리적 모순 검출!

추론 규칙 실습

가족 관계 추론 예제

# 기본 관계 정의
부모(?x, ?y) ← 아버지(?x, ?y)
부모(?x, ?y) ← 어머니(?x, ?y)

# 추론 규칙
조부모(?x, ?z) ← 부모(?x, ?y) ∧ 부모(?y, ?z)
형제자매(?x, ?y) ← 부모(?z, ?x) ∧ 부모(?z, ?y) ∧ 다름(?x, ?y)
삼촌(?x, ?z) ← 형제(?x, ?y) ∧ 부모(?y, ?z) ∧ 남성(?x)
이모(?x, ?z) ← 자매(?x, ?y) ∧ 부모(?y, ?z) ∧ 여성(?x)

# 대칭 관계
형제자매(?x, ?y) → 형제자매(?y, ?x)
배우자(?x, ?y) → 배우자(?y, ?x)

# 이행 관계
조상(?x, ?z) ← 부모(?x, ?y) ∧ 조상(?y, ?z)

2.6 온톨로지 설계 원칙

효과적인 온톨로지를 만들기 위한 핵심 원칙과 모범 사례를 알아봅니다.

설계 원칙

1. 명확성 (Clarity)

의미가 명확하고 모호하지 않게 정의

✅ 좋은 예:
생물학적부모: 유전적 부모
❌ 나쁜 예:
부모: (생물학적? 법적? 양육?)

2. 일관성 (Consistency)

전체 온톨로지에서 일관된 규칙 적용

• 명명 규칙 통일
• 관계 방향 일관성
• 속성 타입 일관성

3. 확장성 (Extensibility)

새로운 개념 추가가 용이한 구조

• 적절한 추상화 수준
• 모듈화된 설계
• 버전 관리 고려

4. 최소 온톨로지 약속

필요한 최소한의 제약만 정의

• 과도한 제약 피하기
• 도메인 특성 고려
• 유연성 확보

온톨로지 설계 프로세스

1

도메인과 범위 정의

• 대상 영역 명확화

2

기존 온톨로지 검토

3

주요 용어 나열

• 핵심 개념 추출
• 동의어 정리
• 용어 정의

4

클래스 계층 구성

• 상하위 관계 정의
• 분류 체계 구축
• 다중 상속 검토

5

속성 정의

• 객체 속성 정의
• 데이터 속성 정의
• 도메인/레인지 설정

6

제약사항 추가

대학 온톨로지 설계하기

다음 요구사항을 만족하는 대학 온톨로지를 설계하세요:

  1. 주요 클래스 정의: 사람(교수, 학생, 직원), 조직(학과, 대학), 과목, 건물
  2. 관계 정의: 소속, 수강, 강의, 위치
  3. 제약사항 추가:
    • 학생은 최소 1개 이상의 과목을 수강해야 함
    • 교수는 정확히 1개의 학과에 소속됨
    • 강의실은 1개의 건물에만 위치함
  4. 추론 규칙 정의:
    • 같은 과목을 수강하는 학생들은 '동기'
    • 학생을 지도하는 교수는 '지도교수'
힌트: 클래스 계층구조를 먼저 그려보고, 각 클래스의 주요 속성을 정의한 후 관계를 연결하세요.
