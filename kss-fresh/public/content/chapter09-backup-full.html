9.1 추론의 철학과 원리

🤔 추론이란 무엇인가?

고대 그리스의 아리스토텔레스부터 현대의 AI까지, 인류는 항상 "알려진 것에서 알려지지 않은 것을 도출하는" 방법을 추구해왔습니다.

"모든 사람은 죽는다. 소크라테스는 사람이다. 따라서 소크라테스는 죽는다." - 삼단논법의 고전적 예시

🎯 추론의 종류

연역적 추론 (Deductive)

🔽

일반적 원칙에서 특수한 결론 도출

예시:
모든 포유류는 온혈동물이다 →
고래는 포유류다 →
∴ 고래는 온혈동물이다

귀납적 추론 (Inductive)

🔼

특수한 관찰에서 일반적 규칙 도출

예시:
백조 1은 희다 →
백조 2는 희다 →
∴ 모든 백조는 희다 (개연적)

귀추적 추론 (Abductive)

🔄

관찰로부터 최선의 설명 추측

예시:
잔디가 젖어있다 →
비가 오면 잔디가 젖는다 →
∴ 비가 왔다 (추측)

🌐 온톨로지 추론의 특징

전통적 데이터베이스

  • Closed World Assumption
  • 모르는 것은 거짓
  • 명시된 데이터만 처리
  • 예: SQL NOT EXISTS

온톨로지 추론

  • Open World Assumption
  • 모르는 것은 가능성 열어둠
  • 암묵적 지식 도출
  • 예: 명시되지 않아도 추론

💡 실제 추론 예시

시나리오: 가족 관계 추론

명시된 사실:
  • John hasChild Mary
  • Mary hasChild Tom
  • hasChild는 transitive 속성
추론된 지식:
  • John hasChild Tom (손자 관계)
  • Mary hasParent John (역관계)
  • Tom hasAncestor John (조상 관계)
추론 과정:
1. hasChild(John, Mary) ∧ hasChild(Mary, Tom)
2. transitive(hasChild)
3. ∀x,y,z: hasChild(x,y) ∧ hasChild(y,z) → hasChild(x,z)
4. ∴ hasChild(John, Tom)

9.2 Description Logic - 추론의 수학적 기초

🧮 Description Logic의 탄생

1980년대, AI 연구자들은 지식 표현과 추론의 균형을 찾고 있었습니다.

"우리는 표현력은 풍부하지만 결정 불가능한 언어와, 결정 가능하지만 표현력이 제한된 언어 사이에서 선택해야 했습니다." - Ronald J. Brachman, DL 창시자

📚 DL 구문과 의미론

DL 구문 의미 OWL 표현 예시
기본 개념
⊤ 전체 클래스 owl:Thing 모든 객체
⊥ 공집합 owl:Nothing 존재하지 않음
뺔리 연산자
C ⊓ D 교집합 intersectionOf Student ⊓ Employee
C ⊔ D 합집합 unionOf Cat ⊔ Dog
¬C 여집합 complementOf ¬Minor
양화사
∃r.C 존재 한정사 someValuesFrom ∃hasChild.Doctor
∀r.C 전칭 한정사 allValuesFrom ∀hasChild.Happy
≥n r.C 최소 기수제약 minCardinality ≥2 hasWheel.Wheel
≤n r.C 최대 기수제약 maxCardinality ≤1 hasMother.Person

🎯 DL 언어 계열

AL (Attributive Language)

⊤, ⊥ A (atomic concepts) C ⊓ D ¬A (atomic negation) ∃r.⊤ ∀r.C

ALC (AL + Complement)

AL + ¬C (full negation) C ⊔ D ∃r.C

SHIQ

ALC + Transitive roles Role hierarchy Inverse roles Qualified cardinality

SROIQ (OWL 2 DL)

SHIQ + Role chains Nominals Datatypes Role assertions

🧬 추론 알고리즘 심화

Tableau 알고리즘

  1. 초기화: ABox에 개체와 개념 추가
  2. 확장: 규칙 적용하여 새로운 사실 도출
    • ∩-rule: C ∩ D → C, D
    • ⊔-rule: C ⊔ D → C 또는 D (분기)
    • ∃-rule: ∃R.C → 새 개체 b, R(a,b), C(b)
    • ∀-rule: ∀R.C, R(a,b) → C(b)
  3. 충돌 검사: {A(x), ¬A(x)} 같은 모순 확인
  4. 종료: 모든 분기가 충돌하면 unsatisfiable
예시: Parent ⊑ Person 검증
1. 가정: Parent(a), ¬Person(a)
2. Parent ≡ Person ∩ ∃hasChild.Person 적용
3. Person(a) ∩ ∃hasChild.Person(a) 도출
4. 충돌: Person(a) ∧ ¬Person(a)
5. 결론: Parent ⊑ Person 참

9.3 클래스 작업하기

클래스 생성과 편집

새 클래스 생성

  1. Classes 탭 선택
  2. 상위 클래스 선택 (기본: owl:Thing)
  3. Add subclass 버튼 클릭 또는 Ctrl+Alt+C
  4. 클래스 이름 입력
예시: 
Thing
└── Vehicle
    ├── Car
    ├── Bicycle
    └── Airplane

클래스 설명 추가

SubClass Of

필요조건 정의

Car SubClassOf hasWheels exactly 4
Equivalent To

필요충분조건 정의

Parent EquivalentTo Person and hasChild some Person
Disjoint With

상호배타적 클래스

Male DisjointWith Female

클래스 표현식 만들기

교집합 (Intersection)

Manchester 구문: Person and hasAge some int[>=18]

의미: 18세 이상인 사람

합집합 (Union)

Manchester 구문: Cat or Dog

의미: 고양이 또는 개

여집합 (Complement)

Manchester 구문: not Vegetarian

의미: 채식주의자가 아닌 것

속성 제한

Manchester 구문: hasChild only Female

의미: 딸만 있는 경우

9.4 속성 작업하기

Object Properties

속성 생성

  1. Object properties 탭 선택
  2. Add sub property 클릭
  3. 속성 이름 입력 (예: hasParent)
  4. 도메인과 레인지 설정

속성 특성 설정

Functional

하나의 값만 가능

hasBirthDate
Inverse Functional

값이 유일함

hasSSN
Transitive

이행적 관계

hasAncestor
Symmetric

대칭적 관계

marriedTo
Asymmetric

비대칭적 관계

parentOf
Reflexive

반사적 관계

knows

Data Properties

데이터 속성 예시

  • hasName: xsd:string
  • hasAge: xsd:integer
  • hasBirthDate: xsd:dateTime
  • hasEmail: xsd:string[pattern "^.+@.+$"]

데이터타입 제약

hasAge some xsd:integer[>= 0, <= 150]
hasGPA some xsd:float[>= 0.0, <= 4.0]
hasStatus some {"active", "inactive", "pending"}

9.5 개체(Individuals) 작업

개체 생성과 관리

개체 생성 단계

  1. Individuals by class 뷰 선택
  2. 해당 클래스 선택
  3. Add individual 클릭
  4. 개체 이름 입력
  5. 속성 값 추가

예시: 사람 개체

Individual: john_doe
    Types: Person
    Facts:
        hasName "John Doe"
        hasAge 35
        hasEmail "john@example.com"
        worksFor acme_corp
        knows jane_doe

단언(Assertion) 유형

Object Property Assertions

개체 간의 관계

john_doe knows jane_doe
Data Property Assertions

개체의 데이터 값

john_doe hasAge 35
Same/Different Individuals

개체 동일성/상이성

john SameAs john_doe
