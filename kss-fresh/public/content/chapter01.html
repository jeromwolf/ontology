1.1 철학에서 컴퓨터과학으로: 온톨로지의 여행

철학의 뿌리: 존재론의 탄생

온톨로지(Ontology)라는 단어를 처음 들으면 낯설고 어려워 보입니다. 하지만 이 개념은 우리 일상과 밀접한 관련이 있습니다.

온톨로지는 그리스어 'ontos'(존재)와 'logia'(학문)가 결합된 말로, 원래는 "무엇이 존재하는가?"를 탐구하는 철학 분야였습니다.

핵심 개념

존재론(Ontology): 세상에 존재하는 것들의 본질과 그들 간의 관계를 탐구하는 학문

  • 형이상학적 존재론: 존재 자체의 본질 탐구
  • 범주적 존재론: 존재하는 것들의 분류와 체계화
  • 형식적 존재론: 모든 영역에 적용 가능한 보편적 구조
  • 영역 존재론: 특정 분야의 개념과 관계 정의

온톨로지의 역사적 여정 (인터랙티브)

사례 연구: 플라톤의 이데아론

플라톤은 현실 세계의 모든 것이 완벽한 '이데아(Idea)'의 불완전한 복사본이라고 주장했습니다.

의자의 존재론적 분석
  1. 이데아의 의자: 완벽한 의자의 개념
  2. 목수의 의자: 이데아를 모방한 실제 의자
  3. 화가의 의자: 실제 의자를 그린 그림

이 계층구조는 현대 온톨로지의 클래스-인스턴스 관계의 기원이 되었습니다.

실습 1: 나만의 첫 온톨로지

가족 관계를 표현하는 간단한 온톨로지를 만들어봅시다.


# Turtle 형식으로 작성하는 가족 온톨로지
@prefix family: <http://example.org/family#> .
@prefix rdf: <http://www.w3.org/1999/02/22-rdf-syntax-ns#> .
@prefix rdfs: <http://www.w3.org/2000/01/rdf-schema#> .

# 클래스 정의
family:Person rdf:type rdfs:Class ;
    rdfs:label "사람"@ko .

family:Male rdfs:subClassOf family:Person ;
    rdfs:label "남성"@ko .

family:Female rdfs:subClassOf family:Person ;
    rdfs:label "여성"@ko .

# 속성 정의
family:hasParent rdf:type rdf:Property ;
    rdfs:domain family:Person ;
    rdfs:range family:Person ;
    rdfs:label "부모를 가짐"@ko .

family:hasChild rdf:type rdf:Property ;
    rdfs:inverseOf family:hasParent ;
    rdfs:label "자녀를 가짐"@ko .

# 인스턴스 예제
family:김철수 rdf:type family:Male ;
    family:hasParent family:김영희 .

family:김영희 rdf:type family:Female ;
    family:hasChild family:김철수 .
연습 문제:
  1. 형제/자매 관계를 나타내는 속성을 추가해보세요.
  2. 조부모-손자 관계를 추론할 수 있는 규칙을 생각해보세요.
  3. 결혼 관계를 표현하는 방법을 설계해보세요.

1.2 온톨로지의 정의와 특징

컴퓨터 과학에서의 온톨로지 정의

"온톨로지는 특정 영역에 존재하는 개념들과 그들 간의 관계를 명시적으로(explicit) 그리고 형식적으로(formal) 표현한 것이다." - Tom Gruber, 1993

온톨로지의 핵심 구성 요소

클래스(Classes)

개념이나 사물의 집합

예시:
  • 동물, 포유류, 개
  • 자동차, 승용차, 전기차
  • 질병, 감염병, 바이러스성질환

인스턴스(Instances)

클래스에 속하는 개별 객체

예시:
  • 우리집 강아지 '뽀삐'
  • 테슬라 모델 3 (VIN: 5YJ3...)
  • COVID-19

관계(Relations)

개념들 간의 연결

예시:
  • is-a (계층 관계)
  • part-of (부분-전체)
  • causes (인과 관계)

속성(Properties)

객체의 특성

예시:
  • 나이, 무게, 색상
  • 최고속도, 연비
  • 증상, 전염성

공리(Axioms)

항상 참인 명제

예시:
  • 모든 개는 포유류다
  • 전기차는 배기가스를 배출하지 않는다
  • 바이러스는 숙주 없이 번식할 수 없다

규칙(Rules)

추론을 위한 논리

예시:
  • A가 B의 부모이고 B가 C의 부모면, A는 C의 조부모다
  • 증상이 3개 이상 일치하면 특정 질병 의심

온톨로지 구성요소 탐색기

온톨로지 vs 다른 지식 표현 방법

특징 온톨로지 데이터베이스 스키마 분류체계(Taxonomy) 시소러스(Thesaurus)
주요 목적 지식 공유와 추론 데이터 저장과 검색 계층적 분류 용어 관계 정의
표현력 매우 높음 (복잡한 관계) 중간 (구조화된 데이터) 낮음 (계층 관계만) 중간 (동의어, 상하위어)
추론 능력 강력한 추론 지원 제한적 (SQL 쿼리) 단순 상속 용어 확장
형식성 형식 논리 기반 관계형 모델 트리 구조 그래프 구조
사용 예시 의료 진단, AI 추론 업무 시스템, ERP 생물 분류, 도서 분류 검색 엔진, 도서관

실습 2: 온톨로지 vs 데이터베이스

같은 정보를 데이터베이스와 온톨로지로 표현하는 차이를 알아봅시다.

데이터베이스 (SQL)

-- 테이블 생성
CREATE TABLE Person (
    id INT PRIMARY KEY,
    name VARCHAR(100),
    age INT,
    job_id INT
);

CREATE TABLE Job (
    id INT PRIMARY KEY,
    title VARCHAR(100),
    department VARCHAR(100)
);

-- 데이터 삽입
INSERT INTO Person VALUES (1, '김철수', 30, 1);
INSERT INTO Job VALUES (1, '개발자', 'IT부서');

-- 조회
SELECT p.name, j.title 
FROM Person p 
JOIN Job j ON p.job_id = j.id;
온톨로지 (OWL/RDF)

# 클래스 정의
:Person rdf:type owl:Class .
:Job rdf:type owl:Class .
:Developer rdfs:subClassOf :Job .

# 속성 정의
:hasJob rdf:type owl:ObjectProperty ;
    rdfs:domain :Person ;
    rdfs:range :Job .

:worksIn rdf:type owl:ObjectProperty ;
    rdfs:domain :Job ;
    rdfs:range :Department .

# 추론 규칙
:Developer rdfs:subClassOf 
    [ rdf:type owl:Restriction ;
      owl:onProperty :worksIn ;
      owl:hasValue :ITDepartment ] .

# 인스턴스
:김철수 rdf:type :Person ;
    :hasJob :개발자직무 .

:개발자직무 rdf:type :Developer .
# 자동 추론: 김철수는 IT부서에서 일한다
주요 차이점:
  • 스키마 유연성: 온톨로지는 런타임에 새로운 관계와 속성 추가 가능
  • 추론 능력: 온톨로지는 명시하지 않은 사실도 추론 가능
  • 의미 표현: 온톨로지는 개념의 의미와 제약사항을 더 풍부하게 표현
  • 상호운용성: 온톨로지는 표준화된 형식으로 시스템 간 공유 용이

1.3 왜 온톨로지가 필요한가?

현대 정보 시스템의 문제점

의미의 모호성

같은 단어가 다른 의미로 사용되는 문제

예시: "Bank"
  • 금융기관 (은행)
  • 강둑
  • 저장소 (데이터 뱅크)

데이터 사일로

시스템 간 데이터 통합의 어려움

예시: 병원 시스템
  • 진료 기록: "환자번호"
  • 검사 시스템: "수진자ID"
  • 약국 시스템: "처방대상자"

지능적 검색의 한계

단순 키워드 매칭의 한계

예시: 검색 쿼리
  • "저렴한 차" → 자동차? 차(tea)?
  • "파리 여행" → 프랑스? 곤충?

온톨로지가 제공하는 해결책

1. 명확한 의미 정의

각 개념의 정확한 의미와 범위를 명시적으로 정의


:FinancialBank rdf:type owl:Class ;
    rdfs:label "금융기관"@ko ;
    rdfs:comment "돈을 예치하고 대출하는 기관"@ko ;
    owl:disjointWith :RiverBank .

:RiverBank rdf:type owl:Class ;
    rdfs:label "강둑"@ko ;
    rdfs:comment "강이나 하천의 가장자리"@ko .

2. 지식 공유와 재사용

표준화된 형식으로 지식을 공유하고 재사용 가능

온톨로지 통합 예시

서로 다른 시스템이 공통 온톨로지를 통해 소통

3. 자동 추론

명시되지 않은 지식도 논리적으로 추론

추론 예시
사실 1: 모든 포유류는 온혈동물이다
사실 2: 고래는 포유류다
추론: 고래는 온혈동물이다

4. 일관성 검증

지식베이스의 논리적 일관성 자동 검증

모순 검출 예시

# 모순되는 정의
:Bird rdf:type owl:Class ;
    rdfs:subClassOf :CanFly .

:Penguin rdf:type owl:Class ;
    rdfs:subClassOf :Bird ;
    rdfs:subClassOf :CannotFly .

1.4 온톨로지의 실생활 예시

우리 주변의 숨은 온톨로지들

시나리오 1: 스마트폰 음성 비서

"내일 오전 10시에 치과 예약 잡아줘"
온톨로지 기반 처리 과정:
  1. 개체 인식: "내일" → 날짜 개체
  2. 시간 추론: "오전 10시" → 10:00 AM
  3. 장소 분류: "치과" → 의료기관 → 병원
  4. 행동 이해: "예약" → 일정 생성 액션
  5. 관계 추론: 사용자 → has_appointment → 치과

:Appointment rdf:type owl:Class .
:MedicalAppointment rdfs:subClassOf :Appointment .
:DentalAppointment rdfs:subClassOf :MedicalAppointment .

:User_Kim :hasAppointment [
    rdf:type :DentalAppointment ;
    :scheduledTime "2024-01-15T10:00:00"^^xsd:dateTime ;
    :location :NearestDentalClinic
] .

시나리오 2: 온라인 쇼핑 추천

상품 검색: "여름 운동화"
1. 개념 확장
  • 여름 → 계절 → 더운 날씨
  • 운동화 → 신발 → 스포츠용품
2. 속성 추론
  • 통기성이 좋은
  • 가벼운 소재
  • 밝은 색상
3. 관련 상품 연결
  • 러닝화
  • 스니커즈
  • 에어 메쉬 신발

시나리오 3: 스마트 네비게이션

목적지 검색: "아이와 갈만한 곳"

:FamilyFriendlyPlace rdf:type owl:Class ;
    owl:equivalentClass [
        owl:intersectionOf (
            :PublicPlace
            [ rdf:type owl:Restriction ;
              owl:onProperty :hasFacility ;
              owl:someValuesFrom :ChildrenFacility ]
            [ rdf:type owl:Restriction ;
              owl:onProperty :safetyLevel ;
              owl:hasValue :High ]
        )
    ] .

# 모든 커피는 카페인을 포함한다
:Coffee rdfs:subClassOf
    [ rdf:type owl:Restriction ;
      owl:onProperty :hasCaffeine ;
      owl:hasValue true ] .

# 디카페인 커피는 예외
:DecafCoffee rdfs:subClassOf :Coffee ;
    rdfs:subClassOf
    [ rdf:type owl:Restriction ;
      owl:onProperty :hasCaffeine ;
      owl:hasValue false ] .

# 라떼는 우유를 포함해야 한다
:Latte rdfs:subClassOf
    [ rdf:type owl:Restriction ;
      owl:onProperty :hasIngredient ;
      owl:someValuesFrom :Milk ] .
도전 과제:
  1. 계절 메뉴를 표현하는 방법을 추가하세요
  2. 사이즈(Tall, Grande, Venti) 개념을 모델링하세요
  3. 세트 메뉴의 관계를 정의하세요
  4. 할인 규칙을 온톨로지로 표현하세요

온톨로지 표현 언어

RDF (Resource Description Framework)

가장 기본적인 온톨로지 표현 언어

특징:
  • 주어-술어-목적어 트리플
  • 그래프 기반 모델
  • 웹 표준 (W3C)

@prefix : <http://example.org/> .

:Pizza123 rdf:type :MargheritaPizza ;
    :hasTopping :Mozzarella ;
    :hasTopping :Tomato ;
    :hasPrice "12000"^^xsd:integer .

RDFS (RDF Schema)

RDF에 클래스와 속성 정의 추가

추가 기능:
  • 클래스 계층구조
  • 속성 도메인/레인지
  • 기본 제약사항

:MargheritaPizza rdfs:subClassOf :Pizza .
:hasTopping rdfs:domain :Pizza ;
    rdfs:range :Topping .

OWL (Web Ontology Language)

가장 표현력이 높은 온톨로지 언어

고급 기능:
  • 복잡한 클래스 표현
  • 카디널리티 제약
  • 논리적 추론

:VeggiePizza owl:equivalentClass [
    rdf:type owl:Class ;
    owl:intersectionOf (
        :Pizza
        [ rdf:type owl:Restriction ;
          owl:onProperty :hasTopping ;
          owl:allValuesFrom :VegetableTopping ]
    )
] .

연습 1: 도서관 온톨로지 설계

도서관 시스템을 위한 온톨로지를 설계하세요.

요구사항:
  • 도서, 저자, 출판사의 관계 표현
  • 도서 분류 체계 (소설, 과학, 역사 등)
  • 대출 관계 모델링
  • 도서 위치 정보
힌트 보기
  • Book, Author, Publisher를 주요 클래스로
  • writtenBy, publishedBy 같은 관계 정의
  • BookCategory를 계층구조로 설계

연습 2: 레스토랑 추천 온톨로지

음식점 추천 시스템을 위한 온톨로지를 구축하세요.

요구사항:
  • 음식 종류별 분류 (한식, 중식, 일식 등)
  • 가격대, 위치, 분위기 속성
  • 사용자 선호도 모델링
  • 메뉴와 재료 관계

연습 3: 스마트홈 온톨로지

IoT 기반 스마트홈을 위한 온톨로지를 만드세요.

요구사항:

  • 방과 기기의 위치 관계
  • 자동화 규칙 표현
  • 에너지 소비 정보
