4.1 RDF란 무엇인가?

RDF(Resource Description Framework)는 웹 상의 자원을 기술하기 위한 W3C 표준 프레임워크입니다. 이는 기계가 이해할 수 있는 형태로 정보를 표현하는 데이터 모델입니다.

핵심 개념

RDF: 웹 자원의 메타데이터를 표현하기 위한 표준 데이터 모델로, 주어-서술어-목적어의 트리플 구조를 사용합니다.

RDF의 탄생 배경

🔴 문제

  • 웹 페이지의 내용을 기계가 이해할 수 없음
  • 정보의 의미가 명확하지 않음
  • 서로 다른 시스템 간 데이터 교환 어려움
→

🟢 해결

  • 표준화된 메타데이터 형식 제공
  • 명시적인 의미 표현
  • 상호운용성 보장

4.2 RDF 트리플: 지식의 원자

RDF의 핵심은 모든 지식을 주어(Subject) - 서술어(Predicate) - 목적어(Object)의 트리플로 표현하는 것입니다.

트리플의 구조

주어 (Subject)

설명하고자 하는 자원

예: 홍길동
→

서술어 (Predicate)

자원의 속성이나 관계

예: 직업은
→

목적어 (Object)

속성 값이나 다른 자원

예: 개발자

실제 트리플 예시

자연어: "세종대왕은 한글을 창제했다"
RDF 트리플:
세종대왕 창제했다 한글
자연어: "서울은 대한민국의 수도이다"
RDF 트리플:
서울 수도이다 대한민국

트리플의 연결: 지식 그래프 구성

여러 트리플이 연결되어 복잡한 지식을 표현

① 이순신 - 직업 - 장군

② 이순신 - 활동시기 - 조선시대

③ 이순신 - 주요업적 - 임진왜란승리

④ 임진왜란 - 발생년도 - 1592년

4.3 URI/IRI: 웹 상의 고유 식별자

RDF에서는 모든 자원을 고유하게 식별하기 위해 URI(Uniform Resource Identifier) 또는 IRI(Internationalized Resource Identifier)를 사용합니다.

URI의 역할

🏷️ 고유 식별

전 세계에서 유일한 이름 부여

http://example.org/people/홍길동

🔗 연결 가능

웹을 통해 접근 가능한 주소

http://dbpedia.org/resource/Seoul

🌍 글로벌 범위

전 세계 어디서나 동일한 의미

http://schema.org/Person

URI 설계 모범 사례

✅ 좋은 예

  • http://data.example.org/person/john-doe
  • http://vocab.example.org/terms/hasName
  • http://id.example.org/book/isbn/978-89-123-4567-8

❌ 피해야 할 예

  • http://example.org/12345 (의미 불명확)
  • http://example.org/temp/data (임시적)
  • http://example.org/person?id=123 (쿼리 파라미터)

4.4 RDF 직렬화 형식

RDF 데이터는 다양한 형식으로 표현될 수 있습니다. 각 형식은 고유한 장단점이 있어 용도에 따라 선택합니다.

주요 직렬화 형식

Turtle (Terse RDF Triple Language)

가장 읽기 쉬운 형식

@prefix ex: <http://example.org/> .
@prefix foaf: <http://xmlns.com/foaf/0.1/> .

ex:홍길동 a foaf:Person ;
    foaf:name "홍길동"@ko ;
    foaf:age 30 ;
    foaf:knows ex:김철수 .
👍 인간 친화적 📝 간결한 문법 🎯 학습 용이

RDF/XML

최초의 W3C 표준 형식

<?xml version="1.0" encoding="UTF-8"?>
<rdf:RDF xmlns:rdf="http://www.w3.org/1999/02/22-rdf-syntax-ns#"
         xmlns:foaf="http://xmlns.com/foaf/0.1/">
  <foaf:Person rdf:about="http://example.org/홍길동">
    <foaf:name xml:lang="ko">홍길동</foaf:name>
    <foaf:age rdf:datatype="http://www.w3.org/2001/XMLSchema#integer">30</foaf:age>
    <foaf:knows rdf:resource="http://example.org/김철수"/>
  </foaf:Person>
</rdf:RDF>
📋 표준 준수 🔧 도구 지원 ⚠️ 장황함

JSON-LD

웹 개발자 친화적 형식

{
  "@context": {
    "foaf": "http://xmlns.com/foaf/0.1/",
    "ex": "http://example.org/"
  },
  "@id": "ex:홍길동",
  "@type": "foaf:Person",
  "foaf:name": {
    "@value": "홍길동",
    "@language": "ko"
  },
  "foaf:age": 30,
  "foaf:knows": {
    "@id": "ex:김철수"
  }
}
🌐 웹 친화적 💻 JavaScript 호환 🔄 REST API

N-Triples

가장 단순한 형식

<http://example.org/홍길동> <http://www.w3.org/1999/02/22-rdf-syntax-ns#type> <http://xmlns.com/foaf/0.1/Person> .
<http://example.org/홍길동> <http://xmlns.com/foaf/0.1/name> "홍길동"@ko .
<http://example.org/홍길동> <http://xmlns.com/foaf/0.1/age> "30"^^<http://www.w3.org/2001/XMLSchema#integer> .
<http://example.org/홍길동> <http://xmlns.com/foaf/0.1/knows> <http://example.org/김철수> .
⚡ 처리 속도 📦 스트리밍 😵 가독성 낮음

4.5 RDF 데이터 타입과 리터럴

리터럴 값의 표현

단순 리터럴

"홍길동"
"Hello World"
"안녕하세요"

언어 태그 리터럴

"홍길동"@ko
"Hong Gildong"@en
"洪吉童"@zh

타입 리터럴

"30"^^xsd:integer
"2024-01-15"^^xsd:date
"true"^^xsd:boolean
"3.14159"^^xsd:double

주요 XSD 데이터 타입

데이터 타입 설명 예시
xsd:string 문자열 "Hello World"
xsd:integer 정수 "42"^^xsd:integer
xsd:decimal 십진수 "3.14"^^xsd:decimal
xsd:boolean 불린 "true"^^xsd:boolean
xsd:date 날짜 "2024-01-15"^^xsd:date
xsd:dateTime 날짜시간 "2024-01-15T10:30:00"^^xsd:dateTime

4.6 RDF 어휘와 네임스페이스

RDF에서는 재사용 가능한 어휘를 정의하고 네임스페이스를 통해 구분합니다.

주요 표준 어휘

FOAF (Friend of a Friend)

사람과 소셜 네트워크 기술

@prefix foaf: <http://xmlns.com/foaf/0.1/>
  • foaf:Person - 사람
  • foaf:name - 이름
  • foaf:knows - 아는 사람
  • foaf:homepage - 홈페이지

Dublin Core

메타데이터 기술

@prefix dc: <http://purl.org/dc/elements/1.1/>
  • dc:title - 제목
  • dc:creator - 작성자
  • dc:date - 날짜
  • dc:subject - 주제

Schema.org

구조화된 데이터 마크업

@prefix schema: <http://schema.org/>
  • schema:Person - 사람
  • schema:Organization - 조직
  • schema:Event - 이벤트
  • schema:Product - 제품

SKOS

지식 조직 체계

@prefix skos: <http://www.w3.org/2004/02/skos/core#>
  • skos:Concept - 개념
  • skos:broader - 상위 개념
  • skos:narrower - 하위 개념
  • skos:related - 관련 개념

4.7 실전 RDF 데이터 작성

예제: 도서관 목록 RDF

시나리오

도서관의 책 정보를 RDF로 표현해봅시다.

Turtle 형식으로 작성
@prefix ex: <http://example.org/library/> .
@prefix dc: <http://purl.org/dc/elements/1.1/> .
@prefix foaf: <http://xmlns.com/foaf/0.1/> .
@prefix xsd: <http://www.w3.org/2001/XMLSchema#> .

# 책 정보
ex:book001 a ex:Book ;
    dc:title "온톨로지 입문"@ko ;
    dc:creator ex:author001 ;
    dc:date "2024-01-15"^^xsd:date ;
    dc:subject "컴퓨터과학", "시맨틱웹" ;
    ex:isbn "978-89-123-4567-8" ;
    ex:pages 350 ;
    ex:price "25000"^^xsd:integer ;
    ex:location ex:shelf_A3 .

# 저자 정보
ex:author001 a foaf:Person ;
    foaf:name "김철수"@ko ;
    foaf:mbox <mailto:kimcs@example.org> ;
    ex:affiliation "한국대학교" .

# 서가 위치
ex:shelf_A3 a ex:Shelf ;
    ex:floor 2 ;
    ex:section "컴퓨터과학" ;
    ex:capacity 100 .
코드 설명
  • 네임스페이스 선언: @prefix로 약어 정의
  • 타입 지정: 'a'는 rdf:type의 약어
  • 속성 나열: 세미콜론(;)으로 구분
  • 다중 값: 쉼표(,)로 구분
  • 언어 태그: @ko로 한국어 명시
  • 데이터 타입: ^^xsd:date로 날짜 타입 지정
