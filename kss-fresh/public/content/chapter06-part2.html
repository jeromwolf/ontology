🔤 Boolean 클래스 표현식

                

                    

                        

                            ⊓
                            교집합 (Intersection)
                        

                        

                            

                                
문법

                                ClassA and ClassB
                            

                            

                                
예제

                                Mother ≡ Woman and Parent
                                
어머니 = 여성이면서 부모인 사람

                            

                            

                                
사용 시나리오

                                
여러 조건을 동시에 만족해야 할 때

                            

                        

                    

                    
                    

                        

                            ⊔
                            합집합 (Union)
                        

                        

                            

                                
문법

                                ClassA or ClassB
                            

                            

                                
예제

                                Pet ≡ Dog or Cat or Bird
                                
애완동물 = 개 또는 고양이 또는 새

                            

                            

                                
사용 시나리오

                                
여러 대안 중 하나를 선택할 때

                            

                        

                    

                    
                    

                        

                            ¬
                            부정 (Complement)
                        

                        

                            

                                
문법

                                not Class
                            

                            

                                
예제

                                NonSmoker ≡ not Smoker
                                
비흡연자 = 흡연자가 아닌 사람

                            

                            

                                
사용 시나리오

                                
제외 조건을 표현할 때

                            

                        

                    

                

            

            
            
            

                
🔗 속성 제한 표현식

                

                    

                        

                            ∃
                            존재 제한 (Existential)
                        

                        

                            

                                
문법

                                property some Class
                            

                            

                                
예제

                                Parent ≡ hasChild some Person
                                
부모 = 적어도 한 명의 사람인 자녀를 가진 사람

                            

                            

                                
인터랙티브 데모

                                

                            

                        

                    

                    
                    

                        

                            ∀
                            전체 제한 (Universal)
                        

                        

                            

                                
문법

                                property only Class
                            

                            

                                
예제

                                Vegetarian ≡ eats only Plant
                                
채식주의자 = 식물만 먹는 사람

                            

                            

                                
⚠️ 주의사항

                                
only는 "만약 있다면"의 의미. 값이 없어도 조건 만족!

                            

                        

                    

                    
                    

                        

                            #
                            카디널리티 제한
                        

                        

                            

                                
문법

                                property min/max/exactly n [Class]
                            

                            

                                
예제들

                                
    
                                    
  • hasChild min 3 - 최소 3명의 자녀
    • 
                                    
  • hasWheel exactly 4 Wheel - 정확히 4개의 바퀴
    • 
                                    
  • hasSpouse max 1 - 최대 1명의 배우자
    • 
                                

                            

                            

                                
카디널리티 실험실

                                
                                3
                                

                            

                        

                    

                

            

            
            
            

                
🎯 고급 클래스 표현식

                

                    

                        

                            ≡
                            값 제한 (HasValue)
                        

                        

                            

                                
문법

                                property value individual
                            

                            

                                
예제

                                KoreanCitizen ≡ hasNationality value Korea
                                
한국 시민 = 국적이 한국인 사람

                            

                        

                    

                    
                    

                        

                            🔄
                            자기 제한 (Self)
                        

                        

                            

                                
문법

                                property Self
                            

                            

                                
예제

                                Narcissist ≡ likes Self
                                
나르시시스트 = 자기 자신을 좋아하는 사람

                            

                        

                    

                    
                    

                        

                            {}
                            열거 (Enumeration)
                        

                        

                            

                                
문법

                                {individual1, individual2, ...}
                            

                            

                                
예제

                                PrimaryColor ≡ {Red, Blue, Yellow}
                                
원색 = 빨강, 파랑, 노랑만 포함

                            

                        

                    

                

            

        

    

    
    
    

        
🎨 실전 OWL 패턴 라이브러리

        

            
        

        
        

            

                
가족 관계 패턴

                

                    
# 부모 정의
Parent ≡ Person and hasChild some Person

# 어머니 정의
Mother ≡ Woman and Parent

# 외동자녀 부모
ParentOfOnlyChild ≡ Parent and hasChild exactly 1 Person

# 다자녀 부모
ParentOfManyChildren ≡ Parent and hasChild min 3 Person
                    

                

                
            

            
            

                
시간 관계 패턴

                

                    
# 현재 활동중
CurrentlyActive ≡ hasStatus value Active 
                 and hasEndDate exactly 0

# 기간 한정
TimeLimited ≡ hasStartDate some DateTime 
            and hasEndDate some DateTime

# 미래 이벤트
FutureEvent ≡ Event and hasStartDate some DateTime
            and hasStartDate > currentDateTime
                    

                

                
            

            
            

                
위치 관계 패턴

                

                    
# 같은 도시 거주자
SameCityResident ≡ Person and livesIn some City
                 and (livesIn exactly 1 City)

# 국제 조직
InternationalOrg ≡ Organization 
                  and hasOfficeIn min 2 Country

# 원격 근무자
RemoteWorker ≡ Employee 
             and (not (workLocation some PhysicalOffice))

🏠 실전 프로젝트: 스마트 홈 온톨로지 구축

        

            

                

                    Step 1
                    
도메인 분석

                

                

                    
스마트 홈의 구성 요소를 파악합니다.

                    

                        

                            🏠
                            Room
                        

                        

                            💡
                            Device
                        

                        

                            👤
                            Resident
                        

                        

                            ⚡
                            Sensor
                        

                    

                

            

            
            

                

                    Step 2
                    
클래스 계층 설계

                

                

                    
Thing
├── Location
│   ├── Room
│   │   ├── Bedroom
│   │   ├── Kitchen
│   │   └── LivingRoom
│   └── Floor
├── Device
│   ├── Actuator
│   │   ├── Light
│   │   ├── Thermostat
│   │   └── DoorLock
│   └── Sensor
│       ├── MotionSensor
│       ├── TemperatureSensor
│       └── SmokeSensor
└── Agent
    ├── Human
    └── AutomationRule
                    

                

            

            
            

                

                    Step 3
                    
속성 정의

                

                

                    

                        

                            
Object Properties

                            
    
                                
  • locatedIn - Device가 위치한 Room
    • 
                                
  • controls - Device가 제어하는 대상
    • 
                                
  • triggeredBy - 자동화 트리거
    • 
                            

                        

                        

                            
Data Properties

                            
    
                                
  • hasTemperature - 현재 온도
    • 
                                
  • isActive - 활성화 상태
    • 
                                
  • powerConsumption - 전력 소비량
    • 
                            

                        

                    

                

            

            
            

                

                    Step 4
                    
복잡한 클래스 정의

                

                

                    

                        

                            
에너지 효율적인 방

                            
EnergyEfficientRoom ≡ Room 
    and hasDevice only LowPowerDevice
    and hasTemperatureSensor min 1
                            
                        

                        

                            
자동화된 방

                            
AutomatedRoom ≡ Room
    and hasDevice some SmartDevice
    and hasMotionSensor some MotionSensor
    and hasAutomationRule min 1
                            
                        

                    

                

            

            
            

                

                    Step 5
                    
추론 규칙 활용

                

                

                    

                        
자동 추론 예시

                        

                            
입력: BathroomLight locatedIn MasterBathroom

                            
추론: BathroomLight는 Device이고, MasterBathroom은 Room

                        

                        

                            
규칙: 모든 방에 연기 감지기가 있으면 "안전한 집"

                            
SafeHome ≡ Home and 
    (hasRoom only (Room and hasSmokeSensor min 1))
