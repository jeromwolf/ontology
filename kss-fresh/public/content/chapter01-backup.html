1.1 철학에서 컴퓨터과학으로: 온톨로지의 여행

철학의 뿌리

온톨로지(Ontology)라는 단어를 처음 들으면 낯설고 어려워 보입니다. 하지만 이 개념은 우리 일상과 밀접한 관련이 있습니다.

온톨로지는 그리스어 'ontos'(존재)와 'logia'(학문)가 결합된 말로, 원래는 "무엇이 존재하는가?"를 탐구하는 철학 분야였습니다.

핵심 개념

존재론(Ontology): 세상에 존재하는 것들의 본질과 그들 간의 관계를 탐구하는 학문

현대적 정의

컴퓨터 과학에서의 온톨로지

특정 분야의 개념들과 그들 사이의 관계를 명확하고 형식적으로 정의한 명세서

  • 개념(Concepts): 무엇이 있는지 예: 사람, 동물, 조직
  • 속성(Properties): 어떤 성질을 가지는지 예: 이름, 나이, 색상
  • 관계(Relations): 서로 어떻게 연결되는지 예: 부모이다, 소속된다, 소유한다
  • 규칙(Rules): 어떤 제약이 있는지 예: 사람은 동시에 두 곳에 있을 수 없다

철학적 온톨로지 vs 컴퓨터 온톨로지

1.2 일상 속 온톨로지 찾기

온톨로지는 추상적인 개념처럼 보이지만, 실제로는 우리 주변 어디에나 있습니다. 우리가 매일 사용하는 분류 체계와 개념 구조가 바로 온톨로지의 예입니다.

1.3 왜 온톨로지가 필요한가?

문제 상황: 정보의 모호성

온톨로지는 각 개념의 의미를 명확히 정의하고, 컴퓨터가 이해할 수 있는 형식으로 표현합니다.

온톨로지를 통한 의미 구분

❌ 온톨로지 없이

- "사과"라는 문자열 매칭
- 문맥 이해 불가
- 의미 구분 불가

결과:
- 모든 "사과" 포함 문서
- 관련 없는 결과 혼재
- 사용자 의도 파악 실패

1.4 온톨로지 vs 데이터베이스 vs 분류체계

온톨로지는 데이터베이스나 단순한 분류체계와는 근본적으로 다릅니다. 각각의 특징과 차이점을 자세히 살펴보겠습니다.

세 가지 접근법의 비교

구분 데이터베이스 분류체계 온톨로지
목적 데이터 저장/관리 체계적 분류 지식 표현/추론
구조 테이블/관계형 계층형 트리 그래프/네트워크

실제 예시: "김철수는 서울에 있는 ABC회사에서 일한다"

데이터베이스 표현

-- 직원 테이블에 삽입
INSERT INTO employees (id, name, company_id) 
VALUES (1, '김철수', 101);

-- 회사 테이블에서 조회
SELECT * FROM companies WHERE id = 101;
-- 결과: {id: 101, name: 'ABC회사', location: '서울'}

⚠️ 한계: 관계의 의미가 명시되지 않음

분류체계 표현

조직
├─ 기업
│  └─ IT기업
│     └─ ABC회사
인력
└─ 직원
   └─ 김철수

⚠️ 한계: 김철수와 ABC회사의 관계 표현 불가

온톨로지 표현

:김철수 rdf:type :Person ;
    :worksFor :ABC회사 ;
    :hasRole "개발자" .

:ABC회사 rdf:type :Company ;
    :locatedIn :서울 ;
    :industry "IT" .

:worksFor rdfs:domain :Person ;
    rdfs:range :Organization .

✅ 장점: 의미와 관계가 명확히 표현되고 추론 가능

🎯 실습 과제

미니 온톨로지 설계하기

자신이 잘 아는 분야를 선택하여 간단한 온톨로지를 설계해보세요.

설계 단계:

  1. 도메인 선택

    예: 커피숍, 영화, 운동, 요리, 음악 등

    💡 자신이 잘 아는 분야를 선택하면 개념과 관계를 정의하기 쉽습니다.
  2. 주요 개념 정의

    최소 5개 이상의 핵심 개념(클래스) 도출

    💡 너무 세부적이거나 너무 포괄적이지 않은 적절한 수준의 개념을 선택하세요.
  3. 속성 정의

    각 개념별 2-3개의 속성 정의

    💡 데이터 속성(값)과 객체 속성(관계)을 구분하여 정의하세요.
  4. 관계 정의

    개념 간 관계를 명확히 정의

    💡 관계의 방향성과 의미를 명확히 표현하세요.

템플릿 예시: 커피숍 온톨로지

1. 주요 개념(클래스)
커피 바리스타 고객 메뉴 매장 원두
2. 속성 정의
커피
  • 데이터 속성: 가격, 용량, 온도
  • 객체 속성: 사용원두, 제조자
바리스타
  • 데이터 속성: 이름, 경력, 자격증
  • 객체 속성: 근무매장, 전문메뉴
3. 관계 정의
바리스타 제조한다 커피
고객 주문한다 메뉴
커피 사용한다 원두
매장 보유한다 메뉴
바리스타 근무한다 매장
4. 규칙 예시
  • 에스프레소 기반 음료는 모두 커피 클래스에 속한다
  • 바리스타는 최소 1개 이상의 매장에 소속되어야 한다
  • 디카페인 커피는 카페인 함량이 0.1% 이하여야 한다
